Principio de diseñoLos módulos en common/ son 100% reutilizables. La lógica de filtrado, análisis y detección de bots es idéntica para ambas estrategias. Solo difiere HOW se buscan los wallets y WHAT se hace con la señal resultante.

La Gamma API (gamma-api.polymarket.com) es pública, sin autenticación. Es el punto de entrada para descubrir mercados y mapear categorías.

Endpoint clave

Uso

Rate limit
~4000 req/10s

Ninguna

Data API — Trades, posiciones, holders

La Data API (data-api.polymarket.com) es pública para lectura. Es donde obtenemos el historial de trades de cada wallet.

EndpointParámetros clavePara qué
GET /activity?user=0x...user (obligatorio), type=TRADE, start, end, limit (max 500), offsetHistórico de trades de un wallet. Paginado.
GET /positions?user=0x...user, sortBy (CURRENT/CASHPNL), limitPosiciones abiertas con P&L
GET /holders?conditionId=0x...conditionId, limitTop holders de un mercado. CLAVE para basket builder.
GET /trades?market=0x...market (conditionId), limitTrades recientes en un mercado. Para scalper pool.
Nota importante — Proxy WalletLos usuarios de Polymarket operan desde un proxy wallet (Gnosis Safe), NO desde su EOA/MetaMask. Cuando trackeas un wallet, debes usar la dirección proxy visible en su perfil de Polymarket. Si usas la EOA, verás 0 trades.

03 Módulo común: wallet_analyzer

El wallet_analyzer recibe trades crudos del Data API y calcula TODAS las métricas necesarias para los filtros. Ambas estrategias lo usan exactamente igual.

Métricas que calcula:

MétricaCálculoUsado por
win_rateMercados con P&L neto positivo / total mercadosTier 1
track_record_daysÚltimo trade - primer tradeTier 1
avg_holding_periodMedia de (último SELL - primer BUY) por mercadoTier 1
pnl_30d / pnl_7dSum(SELL usdc) - Sum(BUY usdc) en ventanaTier 1
trades_per_monthtotal_trades / (track_record_days / 30)Tier 1
profit_factorgross_profit / gross_lossTier 2
edge_vs_oddswin_rate - avg(entry_price). Entry price = implied probability.Tier 2
unique_categoriesHeurística por keywords en slugs/títulosTier 2
positive_weeks_pctSemanas con P&L >0 / total semanasTier 2
interval_cvstd(intervalos) / mean(intervalos) entre tradesBot Test 1
size_cvstd(usdcSizes) / mean(usdcSizes)Bot Test 2
Edge vs Odds — El concepto claveEn Polymarket, el precio de compra ES la probabilidad implícita. Si compras YES a $0.65, el mercado dice que hay 65% de probabilidad. Si tu win rate en ese tipo de mercado es 72%, tu edge real es +7%. Buscamos wallets con edge ≥ +5% sostenido.

04 Módulo común: wallet_filter + bot_detector

Pipeline de filtros (secuencial y eliminatorio)

full_filter_pipeline(metrics):
    # 1. Tier 1 — Si falla uno, DESCARTADO
    if not passes_tier1(m): return False

    # 2. Tier 3 — Alertas de descarte inmediato
    if check_tier3_alerts(m): return False

    # 3. Tier 2 — Debe pasar ≥4 de 6
    if count_tier2_passes(m) < 4: return False

    # 4. Bot detection — Debe pasar ≥4 de 5 tests
    if not passes_bot_tests(m): return False

    return True

Bot detector — Tests avanzados (cross-wallet)

Los tests 1 (interval CV), 2 (size CV) y 5 (frecuencia) se calculan en el analyzer con datos del propio wallet. Los tests 3 y 4 requieren comparar contra OTROS wallets:

Test 3 — Delay Correlation
¿Opera siempre N segundos después de otro wallet?
Para cada mercado en común entre el target y un whale conocido, calcular el delay entre el trade del whale y el trade del target. Si los delays son consistentes (baja varianza, siempre 10-60s), el target es un copier del whale.
Test 4 — Market Originality
¿Opera mercados que los whales no operan?
Calcular qué % de los mercados del target NO aparecen en el pool de whales conocidos. Si es ≥15%, tiene "opinión propia" y probablemente es humano. Si es 0%, todo lo que hace es copiar.

05 Estrategia 1: Basket Builder

Los 3 baskets iniciales

Basket 1 — Más fácil de construir
Crypto Prices
BTC/ETH/SOL price targets con resolución diaria/semanal. Mayor volumen del ecosistema, más wallets con historial largo. CUIDADO: muy dominado por bots — los filtros de §3 son críticos aquí.
Keywords para tags
crypto, bitcoin, btc, eth, solana
Tipo de mercados
"BTC above $X by date"
Resolución típica
1-7 días
Dificultad de filtrado
Alta (muchos bots)
Basket 2 — Mejor ratio señal/ruido
Economía / Fed
Reuniones FOMC, CPI, datos macro. Resolución programada (las fechas de FOMC son públicas). Los traders humanos con edge son analistas macro — más fáciles de distinguir de bots.
Keywords para tags
fed, rate, inflation, cpi, gdp
Tipo de mercados
"Fed rate cut/hold", "CPI above X"
Resolución típica
1-6 semanas (FOMC schedule)
Dificultad de filtrado
Media
Basket 3 — Mayor edge en consenso
Política / Geopolítica
Votos legislativos, executive orders, eventos geopolíticos. El consenso de wallets informados es especialmente valioso aquí porque refleja información privilegiada real (insiders políticos).
Keywords para tags
election, president, congress, vote, tariff
Tipo de mercados
"Bill passes Senate", "Shutdown by X"
Resolución típica
1-14 días
Dificultad de filtrado
Baja (pocos bots)

El motor de consenso monitorea las actividades de todos los wallets del basket y genera señales cuando ≥80% coinciden.

Reglas de salida del Basket:

  • ≥50% del basket cierra → cerrar posición
  • Take profit a 2x return
  • Stop loss al 85% de entrada
  • Timeout 7 días → cerrar a mercado
